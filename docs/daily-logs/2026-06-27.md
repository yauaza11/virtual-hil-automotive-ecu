### 배운것

- CAN은 Data가 0-8 Byte길이 조절해서 보낼 수 있다. 거기서 DLC(data length counter code)가 있는데 이것을 통해 조절가능
- CAN은 Data를 보낼 때 필수적으로 같이 보내야하는 데이터 세 개가 있다.근데 1번항목은 1byte로 한 묶음이다. 이 부분이 2Byte 차지해서 CAN Data는 한 Frame에 최대 8개중 6개로 보낼 수 있다.
    1-1. **Data ID[4 Bit]** : #define CAN_DATA_ID 0x5A 이런거 가져다가 쓰면 된다. 이것과 CAN ID의 다른점은 CAN ID는 HW인식용 및 필터링. DATA ID는 SW 필터링이다.
    1-2. **Sequency Counter[4 Bit]** : 이것은 데이터 유실을 검사하는건데(1,2... '3이 안오네!') 뭐 이런거다. 근데 이게 보통 0-15 범위로 사용하고 4bit만 사용한다.
    
    2. **Checksum/CRC[1 Byte]** : 데이터 결함 확인하는것이고 보통 CRC-8,CRC-16을 씀. data packet 뒤에 바로 붙임
